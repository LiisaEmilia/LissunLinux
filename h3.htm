Tiivistelmä artikkeleista

Name Based Virtual Host eli nimipohjainen virtuaalisivusto.

Mahdollistaa useamman weppi-sivuston ajamisen samalta IP-osoitteelta.

Asetusten määrittämiseen tarvitaan DNS ja APACHE HTTP-palvelimet.

The Apache Software Foundation 2023: Apache HTTP Server Version 2.4 Documentation: Name-based Virtual Host Support. Luettavissa. https://httpd.apache.org/docs/2.4/vhosts/name-based.html. Luettu 9.9.2024.

Asenna ja määritä Web-palvelin

Asenna Apache2: $ sudo apt-get -y install apache2

Korvaa  $ echo "Default"|sudo tee /var/www/html/index.html

Testataan oletussivusto: $ curl localhost $ sudoedit /etc/hosts

a. Testaa, että weppipalvelimesi vastaa localhost-osoitteesta. Asenna Apache-weppipalvelin, jos se ei ole jo asennettuna.

 

b) Etsi lokista rivit, jotka syntyvät, kun lataat omalta palvelimeltasi yhden sivun. Analysoi rivit (eli selitä yksityiskohtaisesti jokainen kohta ja numero, etsi tarvittaessa lähteitä).

c) Etusivu uusiksi. Tee uusi name based virtual host. Sivun tulee näkyä suoraan palvelimen etusivulla http://localhost/. Sivua pitää pystyä muokkaamaan normaalina käyttäjänä, ilman sudoa. Tee uusi, laita vanhat pois päältä. Uusi sivu on hattu.example.com, ja tämän pitää näkyä: asetustiedoston nimessä, asetustiedoston ServerName-muuttujassa sekä etusivun sisällössä (esim title, h1 tai p).

Ensin luodaan hakemisto uudelle name best virtual host, loin index html-tiedoston ja laitoin siihen tekstin hattu.example.com. Muokkasin index html-tiedostoa, VI-editorilla sisällön hattu.example.com.index.html-tiedostoon ja tämän jälkeen loin asetustiedoston. Sen jälkeen enabloidasin eli otin tiedoston käyttöön ja disabloidasin alkuperäisen sekä latasin uudelleen apache2:sen.
